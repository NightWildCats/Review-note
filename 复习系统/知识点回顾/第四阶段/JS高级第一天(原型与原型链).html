数据类型
1. 数据类型的分类 * 基本(值)类型 : Number / String / Boolean / undefined / null * 对象(引用)类型 : Object / Function / Array: 2. 数据类型的判断 * typeof : number / string / boolean / undefined / function * instanceof : object / function / array * === : null / undefined 注意：通过 typeof 判断 null 的类型为Object 3.扩展问题: 1. undefined与null的区别? undefined 未赋值（当变量为定义，但是通过console.log()进行打印，值为undefined） null 赋值 值为null 2. 什么时候给变量赋值为null呢? 总结：应用在函数的开头和结尾以及中间都可以，取决于应用场景 ① 应用在前： 先在进行占位，为一个将要定义的对象设定（但还没确定的对象） ② 函数事件处理：赋值进行函数处理 ③ 应用在后： 空置对象（释放对象），进行对象垃圾回收
数据-变量-内存
数据-变量-内存
1. 什么是数据? * 在内存中可读的, 可传递的保存了特定信息的'东东'（在内存种可读的，可传递的特殊信息） * 一切皆数据, 函数也是数据 * 在内存中的所有操作的目标: 数据 * 算术运算, 逻辑运算 * 赋值 * 参数传递等 2. 什么是变量? ① 在程序运行过程中它的值是允许改变的量，一个变量对应一块小内存, 它的值保存在此内存中 注意：当函数中存在变量，它也存储在栈内存中，函数中的对象属性存储在堆对象中 当一个变量（变量不管怎么样都存储在栈中，根据情况判断其值是基本数据，还是引用数据） （变量为函数中定义的变量）变为对象的时候。会在堆空间独立开辟一块空间（不是在包裹函数中存储）。 3. 什么是内存? ① 内存条==>集成电路板==>通电==>暂时存储正在运行的程序及其产生的数据==>程序停止==> 占用的空间自动释放 ② 一块内存包含2个方面的数据 * 内部存储的数据 * 地址值数据 ③ 内存空间的分类 * 栈空间: 全局变量/局部变量 * 堆空间: 对象/函数/数组 4. 内存,数据, 变量三者之间的关系 * 内存就是一个容器, 用来存储(操作)数据 * 变量是这个容器的标识, 后面我们通过变量来找到内存, 操作其中的数据 5. var a = xxx, a中保存的到底做了什么? * xxx是一个基本数据 * xxx是一个对象 * xxx是一个变量（根据变量的值进行判断）
对象
对象
1. 什么是对象? * 多个数据的集合 * 用来保存多个数据的容器 2. 对象中的数据又是什么? * 属性: 属性名(字符串)和属性值(任意类型) * 方法: 是一个特别的属性(属性值是function) 3. 如何访问对象内部的数据? * .属性名: 易用, 但有时不能用 * ['属性名']: 难写, 但更通用 扩展问题: 1. 为什么要设计[propertyName]方式访问对象属性的语法? 为了适用于特殊的字符：在变量中出现大的”-“、”空格“等 2. 什么时候需要用[propertyName]方式访问对象属性? 属性名不确定的时候，例 ：var 属性名; obj[属性名] 3.对象存储数据相对于变量的好处 便于数据的统一管理，能存储不同类型的数据，方便调用
函数
函数
1.函数的分类： ① 命名函数 function fn() {} ② 匿名函数 function () {} ③ 赋值函数 var fn = function () {} ④ 回调函数：你定义的，你没有调用，但是执行了 回调函数有：DOM事件监听函数，定期器回调函数，ajax回调函数 2. 什么是函数? * 用来实现特定功能的, n条语句的封装体 * 一般使用function来定义 3. 为什么要设计函数语法?使用函数有什么好处? * 提高复用性 * 便于阅读交流 4. 如何理解: 函数也是对象? * 函数有属性: prototype（显示原型） * 函数有方法: call()/apply() (this向括号中的对象) * 可以添加新的属性/方法
原型与原型链
原型与原型链
1. 函数的prototype属性 * 每个函数都有一个prototype属性, 它默认指向一个对象(即称为: 原型对象) * 原型对象中有一个属性constructor, 它指向函数对象; 例子：obj.prototype.constructor===obj 2.显示原型属性与隐式原型属性 ①. 每个函数function都有一个prototype，即显式原型 ②. 每个实例对象都有一个__proto__，可称为隐式原型 ③. 对象的隐式原型的值为其对应构造函数的显式原型的值 总结: ① 函数(构造函数)的prototype属性: 在定义函数时自动添加的, 默认值是一个空对象（空对象是存在默认属性的） ② 实例对象的__proto__属性: 创建对象时自动添加的, 默认值为构造函数的prototype属性值 ③ 程序员能直接操作显式原型, 但不能直接操作隐式原型(ES6之前) 3 原型链 * 访问一个对象的属性时，先在基本属性中查找，如果没有，再沿着__proto__这条链向上找 * 别名: 隐式原型链 * 作用: 查找对象的属性(方法) 4 instanceof是如何判断的? * 表达式: A instanceof B * 如果B函数的显式原型对象在A对象的原型链上, 返回true, 否则返回false 总结：判断 A对象 是否是 B函数 new（创建）的实例
